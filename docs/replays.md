# Replays

Safe replay artifacts are shareable by default. They should contain map/config snapshots, accepted actions, events, state hashes, snapshots where useful, reliability stats, and latency stats.

The CLI writes `replay.safe.json` and `result.json` to the requested output directory. `result.json` is the replay artifact's `result` summary; `--snapshot-interval <ticks>` adds optional hash-only `snapshots` in addition to per-tick `stateHashes`.

Safe replay artifacts use `schemaVersion` `fps-arena-bench.schema.v0.1` with non-empty `matchId`, embedded `config`, embedded `map`, `acceptedActions`, `events`, `stateHashes`, optional hash-only `snapshots`, and `result`. Accepted actions contain nonnegative integer `tick`, configured `contenderId`, schema-valid `action`, and nonnegative `latencyMs`; events contain nonnegative integer `tick`, non-empty `type`, optional configured non-empty `contenderId`, and optional strict `details`; state hashes and snapshots contain nonnegative integer `tick` and `sha256:<64 lowercase hex digest>` `hash`. The replay `matchId` must match the result, the config map id/version must match the embedded map, every configured contender needs a spawn slot, and accepted actions, events, placements, winners, and stats must reference configured contenders.

Event `details` are strict and may only contain finite numeric `amount`, `x`, and `y`; nonnegative integer `ammo` and `invalidActions`; nonnegative `damage`, `health`, and `latencyMs`; and boolean `success` and `timeout`.

Accepted action, event, state hash, and snapshot ticks cannot exceed nonnegative integer `result.ticksElapsed`. Results use `schemaVersion` `fps-arena-bench.schema.v0.1` with required non-empty `matchId`, winner, placements, per-contender stats (`kills`, `deaths`, `damageDealt`, `damageTaken`, `survivalTicks`, `pickupsCollected`), reliability counters (`invalidJson`, `schemaFailures`, `repairAttempts`, `repairSuccesses`, `timeouts`, `fallbackActions`), and latency fields (`averageMs`, `p50Ms`, `p95Ms`, `timeoutBudgetMs`). Stat record keys, placement `contenderId` values, and non-null winner values must be non-empty strings. Stat counters, reliability counters, and latency values are nonnegative where applicable; `timeoutBudgetMs` must be positive. Placement `contenderId` values and positive integer `rank` values must each be unique, ranks must be contiguous from `1`, and a non-null winner must match the rank-1 placement. Placement order is winner first when present, then alive contenders, score, survival ticks, damage dealt, and contender id as the deterministic tie-breaker. `repairSuccesses` cannot exceed `repairAttempts`, and `p50Ms` cannot exceed `p95Ms`.

They must not include raw prompts, raw model outputs, credentials, auth paths, or local environment details. Safe artifact writing validates this with `assertReplaySafe('replay.safe.json', ...)`, which walks strings and rejects unsafe material such as sensitive absolute paths, home-relative paths, bearer or `sk-` tokens, AWS and GitHub tokens, private key blocks, JWTs, and URLs with embedded credentials. Harnesses that prepare diagnostics can use `findUnsafeStrings`/`findUnsafeMatchesInString` to inspect values and `redactValue`, `redactEnv`, or `redactArgs` to replace unsafe strings with `[REDACTED]` before writing private data. Raw diagnostics belong in opt-in private debug artifacts such as `debug.private.jsonl`.

Replay readers validate the artifact schema, rebuild the match from embedded config/map and accepted actions, and compare recorded state hashes plus final `ticksElapsed` and `winner`. Hash mismatches throw `ReplayReconstructionError` by default, or can be returned for diagnostics when mismatch throwing is disabled. Private debug traces are newline-delimited prompt/response/action/error records for local troubleshooting only and are not part of the safe replay artifact.

Generated `replay.safe.json` artifacts can be opened in the local top-down web replay viewer in `apps/web`; see [web-viewer.md](./web-viewer.md) for the full local flow.
