# Configs

Match configs use `schemaVersion` `fps-arena-bench.schema.v0.1` and should pin map version/hash, seed, contenders, ruleset version, timeout budgets, fallback policy, and debug capture settings.

Required fields are `schemaVersion`, `id`, `rulesetVersion`, `map`, `seed`, `maxTicks`, `contenders`, `actionTimeoutMs`, `invalidActionPolicy`, and `capture`.

`id`, `rulesetVersion`, `map.id`, and `map.version` must be non-empty strings. `map` contains `id`, `version`, and a `sha256:<64 lowercase hex digest>` hash for the map file. `seed` must be a nonnegative integer; `maxTicks` and `actionTimeoutMs` must be positive integers. `contenders` must include at least two unique non-empty `id` strings and required non-empty `adapterId` strings, with optional non-empty `displayName`; multiple contenders may reuse the same `adapterId`. `invalidActionPolicy` contains nonnegative integer `maxInvalidActions` and `fallbackAction`, where fallback is either `noop` or `repeat-last-valid`. Provider errors, timeouts, and invalid action schemas use that fallback; `repeat-last-valid` reuses the contender's previous valid action and falls back to `noop` when none exists. Persisted results report these as `schemaFailures`, `timeouts`, and total `fallbackActions`; CLI summaries also expose provider-error and schema-violation counts. `capture` contains `safeReplay` and `privateDebug` booleans; bot-only CLI runs currently always write `replay.safe.json` and `result.json` and do not emit `debug.private.jsonl`.
